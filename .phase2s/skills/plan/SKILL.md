You are entering planning mode. Before writing any code, create a structured implementation plan.

Steps:
1. **Understand**: Read relevant files to understand the current codebase state.
2. **Analyze**: Identify existing patterns, utilities, and conventions to follow.
3. **Plan**: Create a numbered list of specific implementation steps, including:
   - Files to create or modify
   - Functions/classes to add
   - Tests to write
   - Potential risks or edge cases
4. **Review**: Present the plan and ask for feedback before proceeding.

Be specific about file paths and function names. Reference existing code patterns.
